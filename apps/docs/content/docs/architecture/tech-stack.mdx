## Core Stack

| Layer | Technology | Notes |
|-------|-----------|-------|
| Frontend | Next.js 16, React 19, Tailwind CSS v4 | App Router with SSR and RSC |
| Backend | Motia (iii engine), TypeScript step handlers | Rust-based runtime with WebSocket step communication |
| Database | Supabase (PostgreSQL) with Row-Level Security | Transaction mode pooler on port 6543 |
| Auth | Better Auth | Email/password + Google OAuth, session-based |
| Queue | Redis (BullMQ via Motia) | Async event processing, scheduled jobs |
| Real-time | Yjs (CRDT), WebSocket streams | Collaborative editing, live data streaming |
| Monorepo | Turborepo, Bun | Workspace management, fast builds |

## State Management

Veritio uses multiple state management patterns, each for a specific use case:

| Pattern | When to Use |
|---------|-------------|
| **SWR CRUD Factory** | Server data with simple CRUD operations (recommended default) |
| **Raw SWR + Cache Orchestrator** | Complex server queries, pagination, multi-key invalidation |
| **Zustand** | Complex client-side state (always use `skipHydration: true` with `persist()` for SSR compatibility) |
| **Context API** | Component-tree scoped state |
| **useState** | Local component state |

The SWR CRUD Factory (`createCRUDHook` + `createScopedArrayCRUDConfig` from `@/lib/swr/crud-factory`) provides optimistic updates, consistent error handling, automatic cache management, and type-safe mutations out of the box.

For the full state management guide, see the [State Management](/docs/architecture/state-management) page or the [State Management Guide](https://github.com/Ankish8/Veritio/blob/main/docs/STATE_MANAGEMENT_GUIDE.md) in the repository.

## Next.js Configuration Details

### Transpiled Packages

The following packages are forced into the Next.js bundle because Turbopack would otherwise externalize them:

- `pg`
- `pg-pool`
- `@veritio/prototype-test`

### Internationalization

Veritio uses the `next-intl` plugin with configuration at `src/i18n/request.ts`.

### Bundle Analysis

Set `ANALYZE=true` when building to generate bundle analysis reports:

```bash
ANALYZE=true bun run build
```
