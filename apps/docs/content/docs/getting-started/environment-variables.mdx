Copy the environment template and fill in the values:

```bash
cp .env.example apps/veritio/.env.local
```

Open `apps/veritio/.env.local` and configure the variables below. The file is organized into sections by feature area.

## Required Variables

These variables must be set for the application to start:

| Variable | Description | Example / Where to Get It |
|----------|-------------|---------------------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL | `https://xxxxx.supabase.co` -- from Supabase Dashboard, Settings, API |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anonymous key (safe to expose in browser) | `eyJ...` -- from Supabase Dashboard, Settings, API |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key (SECRET, server-only) | `eyJ...` -- from Supabase Dashboard, Settings, API |
| `DATABASE_URL` | PostgreSQL pooler connection string | `postgresql://postgres.xxx:password@region.pooler.supabase.com:6543/postgres` -- from Supabase Dashboard, Settings, Database, Connection String (Transaction Mode) |
| `BETTER_AUTH_SECRET` | Session signing secret (32+ characters) | Generate with: `openssl rand -base64 32` |
| `BETTER_AUTH_URL` | Auth callback base URL | `http://localhost:4001` |
| `NEXT_PUBLIC_APP_URL` | Frontend application URL | `http://localhost:4001` |
| `REDIS_URL` | Redis connection string | `redis://localhost:6379` |

**Important:** The `DATABASE_URL` must use port `6543` (transaction mode pooler), not `5432` (session mode). Transaction mode is required for serverless environments like Vercel and works correctly in local development.

## Real-time Collaboration

To enable Yjs-based collaborative editing in the study builder:

```env
# Master switch
NEXT_PUBLIC_REALTIME_V2_ENABLED=true

# Rollout percentage (0-100) -- set to 100 for local dev
NEXT_PUBLIC_REALTIME_V2_PERCENTAGE=100

# WebSocket URLs
NEXT_PUBLIC_REALTIME_SERVER_URL=ws://localhost:4002
NEXT_PUBLIC_YJS_SERVER_URL=ws://localhost:4002
YJS_SERVER_INTERNAL_URL=http://localhost:4002

# Internal API key for Yjs server auth (optional for local dev)
YJS_INTERNAL_API_KEY=
```

Set `NEXT_PUBLIC_REALTIME_V2_ENABLED=true` and `NEXT_PUBLIC_REALTIME_V2_PERCENTAGE=100` to enable real-time collaboration for all users in local development. See the [Real-time Collaboration](/docs/architecture/realtime) page for details on how the Yjs server works.

## Motia Streams

```env
# Stream WebSocket for real-time assistant streaming
NEXT_PUBLIC_MOTIA_WS_URL=ws://localhost:4004

# Disable Motia's built-in memory server (use Redis instead)
MOTIA_DISABLE_MEMORY_SERVER=true
```

## Live Website Testing

```env
# Cloudflare Worker proxy URL for reverse-proxy mode
# Deploy the worker from /workers directory first
NEXT_PUBLIC_PROXY_WORKER_URL=https://your-proxy-worker.workers.dev
```

The proxy worker must be deployed from the `workers/` directory before using live website testing in reverse-proxy mode. See the [Vercel deployment](/docs/self-hosting/vercel) page for Cloudflare Workers deployment instructions.
